# 创建和管理虚拟通道实例
## 创建实例

1. 进入控制台，点击【全部产品】-【混合组网】-【专线虚拟路由器】，进入产品页面。
2. 在【虚拟通道】tab页面，点击新建虚拟通道按钮。
![](/images/2.png)

3. 在创建页面内填写相关配置，创建虚拟通道并且关联专线虚拟路由器。此页面中填写信息的具体说明如下：

| 配置       | 说明                                                         | 是否必填 |
| :--------- | :----------------------------------------------------------- | :------- |
| 地域       | 选择实例所属的地域                                           | 是       |
| 端口类型   | 选择要创建的端口类型，可选独享/共享，默认为独享。**（见注1）**   | 是       |
| 选择端口    | 选择要与虚拟通道绑定的端口。**（见注2）**                                           | 是       |
| 关联专线虚拟路由器   | 选择要关联的专线虚拟路由器或创建新的专线虚拟路由器                                           | 是       |
| 划分Vlan | 选择是否划分vlan，不划分则所有配置在物理端口生效。当端口类型为“共享时”此项不可操作开关| 是       |
| Vlan ID     | 如选择划分Vlan，则在此处填入对应ID，选择范围为: 2 ~ 4000。当端口类型为“共享时”，仅可选择前期协商确认的ID               | 否       |
| 限速值   | 在此处设置独享端口的期望限速值，共享端口默认为2G                                   | 是       |
| 服务商侧互联网段     | 填入所选的服务商的互联网段                                 | 是       |
| 用户侧互联网段   | 填入所选的互联网段                               | 是       |
| 路由协议   | 选择静态路由或BGP                               | 是       |
| 路由协议优先级   | 在此处定义路由协议的优先等级，**不填代表负载**                               | 否       |
| 目的网段   | 填入客户期望达到的目的网段，可点击加号填写多个                               | 是       |
| 开启健康检查   | 选择是否开启健康检查 **（见注3）**                              | 是       |
| 检查类型   | 选择BFD（Bidirectional Forwarding Detection双向转发检测）或者NQA（Network Quality Analyzer网络质量分析）          | 开启健康检查则是       |
| 监控检查间隔   | 选择检查时间间隔，默认500ms                             | 开启健康检查则是        |
| 健康检查次数   | 选择检查的次数，默认3次                             | 开启健康检查则是        |
| 虚拟通道名称   | 输入自定义的虚拟通道名称                              | 是       |

**注1：独享指：用户接入我司时选择独享一个端口；共享指：指客户通过我司合作伙伴预拉的物理线路接入的方式，物理连接和共享端口由合作伙伴提供，UCloud不保证相关SLA，共享专线适用于无大带宽入云需求、上云时间要求较短的场景。**

**注2：此处可选的端口列表为云下资源接入产品中【虚拟端口】tab页中的虚拟端口实例，如需新建请联系客户经理。共享场景时，您可选择已被共享提供方授权的端口**

**注3：健康检查能持续监测路由相关网络连接状态，确保路由协议（BGP 或静态）下网络通信稳定可靠，及时发现并应对可能的故障。**

   > 配置动态路由时，如需区分不同通道的优先级，以med值作为区分
   >
   > 开启公共服务后，会自动发送和接收公共服务路由，但整体VR遵循LPM规则，如果业务网段与公共服务网段重合，可能影响业务使用

## 删除实例
1. 进入控制台，点击【全部产品】-【混合组网】-【专线虚拟路由器】，进入产品页面。
2. 在【虚拟通道】tab页面，选中目标资源解除与专线虚拟路由器的绑定。
   > 如需删除虚拟通道，需要先解除其与任何PLVR的关联关系。
